# 提交报告（2026-03-15）

## 本次变更
1. 新增统一游戏数据加载 Hook：
   - [frontend/src/hooks/useGameData.js](frontend/src/hooks/useGameData.js)
   - 支持优先加载远程 JSON 数据（含 `manifest`），失败后回退本地静态数据。
   - 增加内存缓存，避免重复请求。
2. 数据图鉴页面改为使用统一数据源：
   - [frontend/src/pages/DataLibrary.js](frontend/src/pages/DataLibrary.js)
   - 移除页面内静态数据直接引用，改为使用 `useGameData()`。
   - 修正 `useMemo` 依赖项，确保数据变化时正确重算。
3. 数据查询页面改为使用统一数据源：
   - [frontend/src/pages/DataQuery.js](frontend/src/pages/DataQuery.js)
   - 移除页面内静态数据直接引用，改为使用 `useGameData()`。
   - `processChartData()` 传入当前数据源，避免使用固定静态数据。
4. 模拟器页面改为使用统一数据源并调整默认防具初始化：
   - [frontend/src/pages/Simulator.js](frontend/src/pages/Simulator.js)
   - 默认头盔/护甲改为在远程数据加载后动态初始化，避免初始时机不一致。
5. 图表处理工具函数参数化：
   - [frontend/src/utils/dataProcessor.js](frontend/src/utils/dataProcessor.js)
   - `processChartData()` 新增 `weapons`、`allModifications` 入参并在函数内构建映射。
6. 同步执行计划文档补充：
   - [DATA_SYNC_EXECUTION_PLAN.md](DATA_SYNC_EXECUTION_PLAN.md)
   - 补充 `modifications.js` 数据文件。

## 变更说明
- 本次提交重点是前端多页面数据源统一，减少静态数据硬编码并提升远程数据接入一致性。
- 在远程数据不可用时，仍可回退到本地静态数据，保证页面可用性。

## 影响范围
- 影响页面：数据图鉴、数据查询、模拟器。
- 影响模块：图表数据处理与远程数据加载。
- 后端接口与数据库结构无变更。
